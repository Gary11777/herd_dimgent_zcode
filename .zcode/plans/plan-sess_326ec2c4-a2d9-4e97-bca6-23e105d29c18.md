# Dimgent Technologies — Website Implementation Plan

## Overview
Build a modern, multi-page Laravel 13 marketing website with Tailwind CSS v4 + Alpine.js, a Refactoring UI–inspired design system, and a hardened contact form. Content is sourced from `drafts/content/*`; images/logo/PDF from `drafts/pics/*`.

## Stack (already in place)
- Laravel 13 (PHP 8.3), Blade, SQLite
- Tailwind CSS v4 via `@tailwindcss/vite`, Alpine.js 3 (npm dependency already present, not yet imported)
- Vite + `laravel-vite-plugin` (inputs: `resources/css/app.css`, `resources/js/app.js`)
- Sendmail mail transport (driver exists in `config/mail.php`)

## Confirmed decisions
- Contact form recipient: `info@dimgent.com` (via `CONTACT_RECIPIENT` env var)
- Assets: copied into `public/` (no storage symlink)
- Turnstile: graceful fallback (renders + validates only when keys present)
- Logo: placed on a neutral rounded chip in the header

## 6. Routing & Controller
`routes/web.php`:
- `GET /` → HomeController
- `GET /products`, `/services`, `/projects`, `/about`, `/contacts` → simple PageController methods (or closures returning views)
- `POST /contacts` → `ContactController@store` (the hardened form)

I'll create **`app/Http/Controllers/PageController.php`** (GET pages, shares nav/footer data) and **`app/Http/Controllers/ContactController.php`** (handles form submission).

A `View::share` in `AppServiceProvider::boot()` will expose `turnstileSiteKey` to all views.

## 7. Hardened Contact Form
**`ContactController@store`** implements all five layers:

1. **CSRF** — `@csrf` in the form (Laravel default; verified by `VerifyCsrfToken` middleware already in the web group).
2. **Honeypot** — hidden field `company_website` (hidden via CSS + `aria-hidden` + `tabindex=-1`). If filled → silently redirect back as if success (bot bait), no email sent.
3. **Rate limiting** — `throttle:5,1` route middleware (max 5 submissions / min / IP). Custom `RateLimiter::for('contact')` in `AppServiceProvider` keyed by IP for cleaner config.
4. **Server validation** — `$validated = $request->validate([...])` with rules: name (required, max 100), email (required, email, max 150), phone (nullable, max 50), subject (nullable, max 150), message (required, min 10, max 5000). Laravel auto-sanitizes via validation; messages are escaped in Blade via `{{ }}`. Strips control chars and validates email format strictly.
5. **Cloudflare Turnstile** (invisible/managed) — `Anodyne\Turnstile` or **manual lightweight integration** (preferred: no new dependency). I'll add a small `Turnstile` service class + a `turnstile:valid` validation rule + a Blade partial that injects the Cloudflare script and renders the widget only when `TURNSTILE_SITEKEY` is set. Verification happens server-side via Laravel `Http` POST to `https://challenges.cloudflare.com/turnstile/v0/siteverify` with the secret. **Graceful fallback**: if `TURNSTILE_SECRET` is empty, the check is bypassed (so local dev works without keys).

**Delivery** — `Mail::raw()` (or a plain mailable) via `sendmail` transport to `config('mail.contact_recipient')` (`CONTACT_RECIPIENT` env). Uses `MAIL_FROM_ADDRESS`/`MAIL_FROM_NAME` from global config. On success: redirect back with a success flash; on failure: log + generic error.

**Form UX** — Alpine drives submit-button loading state and re-renders Turnstile after validation errors.

## 8. Configuration changes
- **`.env.example`**: add the requested mail block + `CONTACT_RECIPIENT` + `TURNSTILE_SITEKEY`/`TURNSTILE_SECRET` (empty defaults).
- **`.env`** (local): mirror the mail block so the site runs locally.
- **`config/services.php`**: add a `turnstile` key block reading the env vars.

- **`bootstrap/app.php`**: no changes needed (CSRF + throttle already available via middleware).

## Design notes (Refactoring UI principles applied)
- Strong hierarchy: large bold headings, small uppercase "eyebrow" labels, generous line-height.
- One accent color used purposefully (primary buttons, active nav, key icons).
- Cards with subtle borders + soft shadows, never heavy.
- Consistent spacing scale (section padding `py-20`, container `max-w-6xl`).
- Generous whitespace; no more than 3 levels of heading depth per page.
- Limited, intentional iconography per section.
